### 学んだこと・気づき

**DB設計**
- 「画面から考える」がDB設計の基本
- 必ず取得できる項目とできない項目を区別してNULL許可を使い分ける
- 外部キー（office_id等）は他のテーブルのidを参照するカラム
- 他のテーブルから辿れる情報はカラムを重複して持たない（例: mail_logsにoffice_idは不要）

**アーキテクチャ**
- Controller / Logic / Repository を分ける理由は可読性・変更への強さ・関心の分離
- Entity = DBの形、DTO = 画面の形
- DTOを使うとViewに渡すものが1つになり、型が明確になる

**インフラ**
- EC2はパブリックサブネット、RDSはプライベートサブネットに置く
- RDSのセキュリティグループで「EC2からの3306のみ許可」とするとより安全
- t2.microは無料枠対象、無料枠終了後はt3.microが同価格帯で性能が高い

### 完了したこと

- `docker-compose.yml` 作成（MySQL 8.0コンテナ）
- `.env` をSQLiteからMySQLに切り替え

### 学んだこと・気づき

- `.env` の書き換えだけではダメ。接続先のMySQLそのものを用意する必要がある
- DockerはMySQLをプロジェクト単位で管理する手段（Macをグローバルに汚さない）
- `docker compose up -d` の `-d` はバックグラウンド起動（detached mode）
- `ports: "3306:3306"` = Mac側のポート → コンテナ内のポート
- `volumes` を使わないとコンテナ停止のたびにDBのデータが消える
